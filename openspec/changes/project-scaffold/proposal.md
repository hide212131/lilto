## Why

Lilt-AI の開発を始めるにあたり、Tauri をベースとした基本プロジェクト構造が存在しない。まず動作可能な骨格を構築し、ウェイクワード検出・音声入力・AIエージェント統合などの機能開発を進められる土台を作る。

## What Changes

- Tauri + Rust + フロントエンド（HTML/CSS/JS または TypeScript）の初期プロジェクトを作成する
- macOS・Windows 両対応のビルド設定を用意する
- システムトレイ常駐（デスクトップ常駐）の基本動作を実装する
- アプリ起動・終了・トレイアイコンの表示を動作させる
- 機能モジュール（ウェイクワード・音声入力・AIエージェント・スキル・ハートビート）を追加できる拡張可能な構成にする

## Capabilities

### New Capabilities

- `desktop-shell`: Tauri アプリの基本シェル。ウィンドウ管理・システムトレイ常駐・アプリのライフサイクル制御を担う
- `build-config`: macOS・Windows 向けのクロスプラットフォームビルド設定。CI/CD や開発環境のセットアップを含む

### Modified Capabilities

（なし — 既存スペックなし）

## Impact

- **新規ファイル**: `src-tauri/`（Rust バックエンド）、フロントエンドディレクトリ、`tauri.conf.json` など Tauri 標準構成一式
- **依存関係**: Rust ツールチェーン、Node.js（フロントエンドビルド用）、Tauri CLI
- **対応 OS**: macOS（主要ターゲット）、Windows（副次ターゲット）
- **将来の機能への影響**: ウェイクワード検出・音声入力・AIエージェント・スキルランナー・ハートビートはすべてこのスキャフォールドの上に実装される
